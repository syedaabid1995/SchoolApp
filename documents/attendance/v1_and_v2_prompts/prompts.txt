PROMPT V01 - MASTER GUARANTEE PROMPT (USE FIRST)

You are working inside an existing multi-school School SaaS platform.

ARCHITECTURAL CONSTRAINTS:
- Do NOT refactor existing implemented modules (currently: auth, users/roles, schools, subscriptions, messaging, cache, academic masters if already present).
- For modules not yet implemented (attendance/leave/payroll), you may design and add them, but:
  - keep additive changes
  - do not break current routes/contracts
  - follow tenant isolation and role guard patterns
- All work must be additive and backward-compatible.
- Tenant isolation by schoolId is mandatory.

MANDATORY OUTPUT REQUIREMENTS:
For any new feature, screen, or API, you MUST include:
1. Migration plan + rollback plan (if DB changes exist)
2. Permission matrix impact (role × action)
3. Cache impact + audit log impact
4. Test checklist (API, role access, tenant isolation)

Your task is to EXTEND the system safely, not redesign it.
